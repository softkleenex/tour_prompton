## 🧠 Part 3. ennoia (엔노이아) 플랫폼 기능 가이드

ennoia는 LLM을 RAG 문서폴더, API 커넥터/MCP와 연결하여 AI 에이전트를 개발 및 배포할 수 있는 AX 플랫폼입니다.

### 🌐 3대 작업 공간 구성
1.  **Studio (만드는 공간)**: 빌더가 에이전트 및 멀티 에이전트의 프롬프트를 작성하고, RAG 문서폴더 연결, API 커넥터/MCP 연동, 시나리오 테스트를 수행하는 중앙 개발 공간.
2.  **Apps (사용하는 공간)**: Studio에서 배포된 최종 AI 앱을 사용자가 대화창 인터페이스를 통해 사용하는 사용자 공간.
3.  **Admin (운영하는 공간)**: 리소스 관리, 권한 부여, 토큰 사용량 실시간 모니터링 로그 분석을 담당하는 관리 공간.

### 🧠 에이전트 세부 빌딩 구성 요소
*   **시스템 메시지 (System Message)**: 에이전트에게 역할(Persona), 핵심 작업 임무(Instruction), 그리고 반드시 지켜야 할 제약 사항(Constraints)을 주입하는 영역.
*   **모델 설정 (Temperature & Option)**: 사실 검증 관광 서비스의 경우 **0.3 내외의 저온 설정 권장**. Max Tokens는 일정이 끊기지 않도록 충분히 크게 설정.
*   **RAG (문서폴더) 검색 기법**:
    *   **혼합 검색 (Hybrid Search)**: 유사도 검색과 키워드 검색을 **5:5**로 혼합 연산. 고유명사나 특정 편의시설 정보가 들어맞아야 하는 **관광 데이터 매칭 시 권장**.
    *   **검색 데이터 개수 (Top-K)**: 한 번의 질의에 RAG 문서에서 가져올 텍스트 조각 개수 지정 (보통 **4~6개** 권장).
*   **가변 설정 (Variables)**: 시스템 메시지 내에 `${variable_name}` 형태로 가변 변수를 심어두고, 우측 설정 패널에서 기본값(Default Value)을 지정하여 관리.

## 💬 Part 4. 참가자 문의 종합 FAQ 및 데이터 활용 규정

### 1. 계정 가입 및 권한/접속 오류
*   **초대 링크 오류 및 권한 없음 (승인 대기)**:
    *   메일 본문 주소 맨 끝에 대괄호(`]`)가 포함되었는지 확인하고 제거하여 접속하십시오.
    *   선가입 후 권한 연동이 안 된 경우, 디스코드 스태프에게 `[팀명 / 팀장명 / 가입 이메일]`을 디스코드 DM으로 보내주시면 수동 승인 처리됩니다.
*   **팀원 소속 입력**: 팀원 개별 가입 시 소속란에는 학교/회사명 대신 공모전에 등록한 **`[팀명]`을 정확히 입력**해야 명단 매칭이 가능합니다.

### 2. 엔노이아(Ennoia) 툴 기술 가이드
*   **앱 빌드 시 "에이전트 없음" 오류**: 에이전트 설계 화면 오른쪽 테스트 창의 대화 기록(채팅 로그)을 완전히 초기화(Clear)한 상태에서 저장 및 배포를 완료해야 앱 빌드 시 정상 인식됩니다.
*   **대화 기록 초기화**: 설계 화면 우측 테스트 창에서 대화 초기화 버튼을 클릭한 뒤 순수 템플릿 상태로 다시 배포하십시오.
*   **공모전 사용 에이전트 유형 제한**: 본 대회 환경에서는 **"에이전트" 탭과 "멀티 에이전트" 탭 두 가지만 사용하여 결과물을 제작**해야 합니다. (커스텀 에이전트 및 특화 UI 사용 지양)
*   **음성 및 파일 생성**: 공모전용 엔노이아 플랫폼에서는 음성 출력(TTS) 및 직접적인 파일 생성/다운로드 기능은 기술적으로 지원하지 않습니다.

### 3. 데이터 활용 및 API 연동 범위 규정
*   **한국관광공사 OpenAPI 활용**: API Key는 공공데이터포털(data.go.kr)에서 참가자가 직접 신청하여 발급받아야 합니다.
*   **데이터 활용 원칙 (중요 규정)**:
    *   **한국관광공사 데이터**: **실시간 OpenAPI 호출 필수**. API 호출 없이 정적 파일 데이터(CSV, JSON 등)만 탑재하여 에이전트를 구동하는 방식은 **불인정**됩니다. 단, 성능 안정을 위한 보조적 캐싱은 소명 시 인정됩니다.
    *   **외부/민간 데이터**: 타 기관 데이터(기상청 등)나 민간 지도 데이터는 **RAG 파일 형태(CSV, JSON) 및 외부 API 형태로 자유롭게 혼합 활용 가능**합니다. (크리넥스의 기상청 API 연동 및 RAG 탑재는 100% 규정에 적합합니다.)

### 4. 제출물 및 중복 참여
*   **프론트엔드 웹 화면 구축**: 자체 프론트엔드 웹/앱 구현은 참가자 자율 사항으로 필수 제출이 아니며, 미제출로 인한 감점은 없습니다.
*   **중복 응모 불가**: 동일한 아이디어/서비스로 공모전 내 타 부문(웹·앱 개발/고도화 등)에 중복 지원할 수 없으며, 적발 시 심사 배제됩니다.
