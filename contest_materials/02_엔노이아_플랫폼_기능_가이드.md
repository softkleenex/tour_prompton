# 📄 ennoia (엔노이아) 플랫폼 기능 마스터 가이드

> **본 문서는 사용자님이 캡처해주신 엔노이아 사용 가이드북 26장 분량의 원시 텍스트 자료를 완벽하게 해독 및 구조화하여 작성한 크리넥스(Kleenex) 팀 전용 플랫폼 활용 기술 가이드입니다.**

---

## 🛠️ 1. ennoia(엔노이아) 플랫폼 개요

ennoia는 거대언어모델(LLM)을 기업 내부 데이터 및 시스템과 유기적이고 안전하게 연결해, 생성형 AI 에이전트와 전용 어플리케이션을 신속하게 개발하고 배포할 수 있는 **AX(AI Transformation) 플랫폼**입니다.

### 🌐 3대 작업 공간 구성
1.  **Studio (만드는 공간)**: 빌더(개발자/기획자)가 에이전트 및 멀티 에이전트의 프롬프트를 작성하고, RAG 문서폴더 연결, API 커넥터/MCP 연동, 시나리오 테스트를 수행하는 중앙 개발 공간.
2.  **Apps (사용하는 공간)**: Studio에서 배포된 최종 AI 앱을 사용자가 대화창 인터페이스를 통해 사용하고, 추천 질문을 던지거나 파일을 첨부해 실업무에 활용하는 사용자 공간.
3.  **Admin (운영하는 공간)**: 프로젝트 리소스 관리, 조직원 권한 부여, AI 토큰 사용량 실시간 모니터링 및 모니터링 로그 분석을 담당하는 전용 관리 공간.

## 🧠 2. 에이전트 (Agent) 세부 빌딩 가이드

에이전트는 Studio에서 구축하는 생성형 AI의 가장 기본 단위입니다. LLM 모델, 프롬프트, 도구를 유기적으로 엮어 특정 미션을 수행하도록 최적화합니다.

### 1) 핵심 구성 패널 및 세팅 요소
*   **시스템 메시지 (System Message)**: 에이전트에게 역할(Persona), 핵심 작업 임무(Instruction), 그리고 반드시 지켜야 할 제약 사항(Constraints)을 주입하는 프롬프트 영역.
*   **모델 선택 및 추론 옵션 (우측 패널)**:
    *   *LLM 모델*: Claude 3.5 Sonnet, GPT-4o, Gemini, HyperCLOVA X 등 서비스 요구사양에 맞는 최적의 브레인을 선택 가능.
    *   *Temperature (온도)*: 창의성과 고정 정보 응답 비율 제어. (사실 검증 관광 서비스의 경우 **0.3 내외의 저온 설정 권장**)
    *   *Max Tokens*: 최대 출력 분량. 일정이 끊기지 않도록 **2500 이상 권장**.
    *   *존재/빈도 패널티*: 동일 패턴의 반복 문구 및 중복 명소 추천을 방지.

### 2) RAG (문서폴더) 탑재 및 지식베이스 설정
에이전트가 특정 도메인 지식(예: 무장애 여행 가이드 PDF)에 기반해 대답하도록 컨텍스트를 주입합니다. 할루시네이션(거짓 정보)을 원천 차단하고 보안 데이터를 안전하게 참조할 수 있습니다.
*   **검색 기법 (Search Strategy)**:
    *   **혼합 검색 (Hybrid Search)**: 단어 유사성(유사도 검색)과 단어 완전 매치(키워드 검색)를 **5:5** 비율로 혼합하여 연산합니다. 고유명사나 특정 정보가 정확히 들어맞아야 하는 **관광 데이터 매칭 시 강력히 권장**됩니다.
    *   **기본 검색**: 100% 코사인 유사도(벡터 임베딩 검색)로만 동작합니다.
*   **검색 데이터 개수 (Top-K)**: 한 번의 질의에 RAG 문서에서 가져올 텍스트 조각 개수 지정 (보통 **4~6개**가 응답 속도와 정합성의 절충안으로 권장).

### 3) 가변 설정 (Variables) 기능
*   시스템 메시지 내에 `${variable_name}` 형태로 가변 변수를 심어두고, 우측 설정 패널의 `가변 설정` 영역에 각 변수의 **기본값(Default Value)**을 지정하여 관리합니다.
*   사용자 입력 폼이나 외부 API에서 동적으로 값을 밀어 넣지 않더라도 설정된 기본값으로 안전하게 구동되므로 테스팅 및 유연한 템플릿 제어에 특화되어 있습니다.

## 🔌 3. 실시간 외부 연동 도구 (API 커넥터 & MCP)

에이전트가 학습되지 않은 실시간 공공데이터나 외부 인프라(Slack, Jira 등)와 실시간 통신하여 데이터를 가져오고 명령을 실행하게 돕는 연계 모듈입니다.

### 1) API 커넥터 (API Connector)
*   외부 REST API(GET/POST/PUT 등)를 에이전트와 직접 연결해 데이터를 실시간 조합합니다.
*   **파라미터 주입**: API의 URL 또는 쿼리 파라미터 값에 `${text}` 나 `${variable}` 형식을 사용하면 에이전트가 대화 컨텍스트에서 변수를 실시간 파싱하여 자동으로 API 파라미터에 전달해 줍니다.
*   **샘플 목록 제공**: 한국관광공사 국문 관광정보, 두루누비 트래킹 코스, 고캠핑 정보 API 등 핵심 공공데이터 연계 템플릿이 이미 내장되어 있어 **클릭 한 번으로 자동 셋업**이 가능합니다.

### 2) MCP (Model Context Protocol)
*   LLM과 외부 다양한 도구들 간의 통신 프로토콜을 단일 표준화한 개방형 프로토콜입니다. 
*   **개인 인증**: `내 계정 ➡️ 연결` 메뉴에서 개별 API Key나 OAuth 인증을 최초 1회만 등록해 두면, 여러 에이전트에서 복잡한 API 스키마를 수동 기입할 필요 없이 클릭 한 번으로 모든 도구를 활성화할 수 있습니다.
*   **한국관광공사 MCP**: 63가지 공사 필수 API 오퍼레이션이 도구 형태로 내장되어 있어, 에이전트가 대화 맥락에 따라 가장 적합한 API 도구를 **자동 호출(Tool Calling)**하여 데이터를 스스로 획득합니다.

## 캔버스 4. 멀티에이전트 (Multi-Agent) 설계 가이드

복잡한 시나리오나 업무 프로세스(분류 ➡️ 상세 조회 ➡️ 정합성 검증 ➡️ 승인 ➡️ 생성)를 GUI 기반 캔버스에서 노드를 이어 완성하는 기능입니다.

### 🔲 주요 연동 노드 명세
*   **`Start (시작 노드)`**: 워크플로우의 실행 진입점. 전역 상태 변수(`state`)를 선언하여 분기 처리 및 카운터 루프 연산의 초기값으로 활용합니다.
*   **`에이전트 노드`**: 미리 스튜디오에서 제작 완료한 특정 단일 에이전트를 노드로 추가하여 역할을 위임합니다.
*   **`오케스트레이터`**: 메인 오케스트레이터 LLM이 사용자의 요구사항을 파악해 캔버스에 배치된 에이전트들 중 어떤 대상을 실행할지 자동으로 선택하여 분배합니다.
*   **`Classify (분류 노드)`**: 기획자가 세부 카테고리와 설명을 예시로 기입해두면, LLM이 입력을 판별하여 정확히 일치하는 카테고리 선상의 에이전트 노드로 하이패스 분기해 줍니다.
*   **`if/else` & `while`**: 상태 변수의 참/거짓 논리값을 기반으로 동선을 분기하거나 조건 충족 시까지 연결된 노드를 반복 실행합니다.
*   **`User approval (사용자 승인)`**: 이 노드를 만나면 챗창에 "승인 / 거절" 팝업이 노출되며, 사용자가 최종 승인을 눌러야만 후속 노드(예: 메일 발송, API 데이터 생성/삭제 등)로 안전하게 진입하여 오동작을 미연에 방지합니다.

## 🚀 5. 최종 앱 배포 및 외부 공유 (산출물 제출 핵심)

개발 완료된 에이전트나 멀티에이전트 워크플로우는 최종 배포 프로세스를 거쳐 공유 가능한 URL 링크로 생성해야 공모전 접수가 가능합니다.

1.  **배포하기**: 스튜디오 상단의 **[배포하기]**를 눌러 최신 에이전트/멀티에이전트 설정을 고정합니다.
2.  **공유 링크 생성**: 앱 목록 우측의 더보기 `...` 버튼 클릭 후 **[공유하기]** 메뉴로 이동합니다.
3.  **설정 및 생성**: 최대 사용자 수 등 기본 권한 설정을 조율한 뒤 **[공유 링크 생성]** 버튼을 클릭합니다.
4.  **최종 제출**: 생성된 **ennoia Studio 앱 공유 URL**을 복사하여 접수처 구글 폼에 제출합니다. 
    *   *핵심 특징*: 이 공유 링크는 **로그인 없이 모든 외부 평가위원이 바로 접속하여 사용**할 수 있어, 기능 심사의 핵심 타겟 페이지가 됩니다.
