# 🌀 크리넥스 멀티 에이전트 시스템 캔버스 설계서 (System Canvas Blueprint)

> [!IMPORTANT]
> **최종 릴리즈 V3.5 캔버스 아키텍처 명세서**  
> 본 문서는 엔노이아(ennoia) Studio 캔버스 워크플로우에 조건부 라우팅(If/Else 분기) 기법을 도입하여 완벽히 구현된 크리넥스(Kleenex) 서비스의 멀티 에이전트 시스템 설계도입니다. 개별 에이전트 객체들을 캔버스 노드로 바인딩하고 이들 간의 앞뒤 연결 관계 및 상태 데이터 변수 흐름을 정의합니다.
> (※ 각 에이전트 노드의 구체적인 내부 프롬프트와 세부 필터 파라미터는 본 문서 후반부의 **4. 에이전트별 상세 스펙 및 마스터 프롬프트** 섹션을 참조하십시오.)

## 🔲 1. 전역 상태 변수 정의 (Global State Variables)

`Start` 노드가 실행될 때 생성되어 전체 에이전트 노드 및 분기 조건에서 공유되는 데이터셋입니다.

*   **`state.user_needs`** (String): 보행 약자 세그먼트 (기본값: `"휠체어 이용 보행 약자"`)
*   **`state.target_region`** (String): 여행 선호 지역 (기본값: `"서울 종로구"`)
*   **`state.weather_status`** (String): 기상 안전 적합도 판정 등급 (기본값: `"NORMAL"`)

## 🧭 2. 최종 배포된 조건부 라우팅 캔버스 흐름 (Dynamic Routing Flow)

기상청 자외선 등급 판정에 따라 안전한 경우에는 `비상_우회관` 호출을 완전히 생략하고 바로 `User approval` 게이트로 진입하는 최적화 경로입니다. 또한, 일정이 반려되는 예외 상황에 대응하기 위해 리셋 프로세스가 탑재되어 있습니다.

```text
[Start 노드]
      │
      ▼
[1. 크리넥스_기상_보안관]  ➡️  [2. 크리넥스_기획관]
                                     │
                                     ▼
                             [3. If/else 조건 노드]
                                     │
                 ┌───────────────────┴───────────────────┐
                 ▼ (True: 날씨 위험)                      ▼ (False/Else: 날씨 안전)
        [4. 크리넥스_비상_우회관]                             │
                 │                                       │
                 └───────────────────┬───────────────────┘
                                     │
                                     ▼
                            [5. User approval 노드]
                                     │
                 ┌───────────────────┴───────────────────┐
                 ▼ (Approve: 승인 시)                      ▼ (Reject: 거절 시)
          [최종 뽀송 일정 출력]                       [6. Set state (변수 리셋)]
                 │                                       │
                 ▼                                       ▼
            [대화 세션 종료]                       [7. 크리넥스_안내관]
                                                         │
                                                         ▼
                                                    [대화 세션 종료]
```

## ⚙️ 3. 노드별 캔버스 연결 및 변수 흐름 (Canvas Configurations)

### 1) `Start` (시작 노드)
*   **이전 연결**: 없음 (전체 대화 워크플로우의 기동점)
*   **이후 연결**: `크리넥스_기상_보안관` 노드로 흐름 연결
*   **캔버스 상태 변수 설정 (State Initialization)**:
    *   `state.user_needs` ➡️ `"휠체어 이용 보행 약자"` 선언 (기획관의 RAG 가이드북 대조용)
    *   `state.target_region` ➡️ `"서울 종로구"` 선언 (OpenAPI 위치 범위 인자용)
    *   `state.weather_status` ➡️ `"NORMAL"` 선언 (기상 등급 전역 공유용)

### 2) `크리넥스_기상_보안관` (에이전트 노드)
*   **이전 연결**: `Start` 노드
*   **이후 연결**: `크리넥스_기획관` 노드
*   **캔버스 데이터 흐름 (Output Mapping)**:
    *   에이전트가 `기상청_자외선지수_조회` 커넥터를 호출하여 도출한 최종 기상 판정 코드(`NORMAL` / `CAUTION` / `EMERGENCY`)를 전역 상태 변수인 **`state.weather_status`**에 덮어써서 기록합니다.

### 3) `크리넥스_기획관` (에이전트 노드)
*   **이전 연결**: `크리넥스_기상_보안관` 노드
*   **이후 연결**: `If/else` 노드
*   **캔버스 데이터 흐름 (Input/Output)**:
    *   **입력**: 전역 변수 `state.user_needs` 및 `state.target_region` 참조.
    *   **출력**: 한국관광공사 MCP 도구를 활용해 1차 2박 3일 무장애 코스 초안(마크다운 표 및 지도 하이퍼링크 등)을 작성하여 대화 컨텍스트에 추가합니다.

### 4) `If/else` (조건 노드)
*   **이전 연결**: `크리넥스_기획관` 노드
*   **이후 연결**:
    *   **True (참)** ➡️ `크리넥스_비상_우회관` 노드
    *   **False/Else (거짓)** ➡️ `User approval` 노드
*   **조건식 (Rule)**:
    ```javascript
    state.weather_status != "NORMAL"
    ```
*   **분기 제어**:
    *   기상 판정이 정상 수준이 아닌 모든 상황(`CAUTION` / `EMERGENCY`)일 때는 `비상_우회관` 노드로 우회 처리합니다.
    *   날씨가 안전한 `NORMAL` 상황인 경우, `비상_우회관` 노드를 원천적으로 스킵(Skip)하여 캔버스 레이턴시를 최소화합니다.

### 5) `크리넥스_비상_우회관` (에이전트 노드)
*   **이전 연결**: `If/else` 노드 (True 분기선)
*   **이후 연결**: `User approval` 노드
*   **캔버스 데이터 흐름 (Input/Output)**:
    *   **입력**: 앞서 `기상_보안관`이 매핑한 `state.weather_status` 값과 `기획관`이 생성한 1차 일정을 입력받아 구동합니다.
    *   **출력**: 야외 일정을 실내 무단차 코스로 자동 우회 대체(Swap)하여 보정된 최종 2박 3일 동선을 도출합니다.

### 6) `User approval` (사용자 승인 노드)
*   **이전 연결**: 
    *   `If/else` 노드 (False/Else 분기선에서 다이렉트 진입)
    *   `크리넥스_비상_우회관` 노드 (우회 보정이 완료된 후 진입)
*   **이후 연결 (두 갈래 분기 제어)**:
    *   **Approve (승인 시)**: 캔버스의 최종 대답 출력 및 대화 세션 종료(End)로 연결됩니다.
    *   **Reject (거절 시)**: 변수 데이터를 리셋하기 위해 **`Set state`** 노드로 연결합니다.
*   **출력 배너 메시지 (복사용)**:
```markdown
🛡️ **[크리넥스 안전 검증 시스템 가동 완료]**  

제안해 드린 일정 후보지들에 대해 한국관광공사 무장애 인프라 데이터(물리 단차 2.0cm 이하 검증) 및 기상청 실시간 기상 안전 지수(자외선 및 체감온도 평가)를 100% 교차 정합성 검증 완료했습니다.  

아래 **[승인(Approve)]** 버튼을 클릭하시면, 크리넥스가 최종 보증하는 뽀송뽀송하고 안전한 2박 3일 맞춤형 무장벽 일정을 화면에 출력해 드립니다.
```

### 7) `Set state` (상태 리셋 노드) ★멀티에이전트 전용 노드
*   **이전 연결**: `User approval` 노드 (Reject 단자)
*   **이후 연결**: `크리넥스_안내관` 노드
*   **작업 내용 (State Reset)**:
    *   `state.weather_status` ➡️ `"NORMAL"` 로 초기화
    *   `state.target_region` ➡️ `""` (빈값)으로 초기화
    *   *이후 사용자가 새 대화를 쳤을 때 기존 기상 등급 및 지역 오염 데이터가 미치는 오작동 영향을 차단합니다.*

### 8) `크리넥스_안내관` (안내원 에이전트 노드)
*   **이전 연결**: `Set state` 노드
*   **이후 연결**: 최종 대화 세션 종료 (End)
*   **버전**: `안내관_1`
*   **역할**: RAG/MCP 도구 없이 가볍게 구동되며, 시스템이 정상 초기화되었음을 정중히 알리는 에이전트 안내 배너를 출력하고 세션을 완료합니다.

## 🧠 4. 에이전트별 상세 스펙 및 마스터 프롬프트 (Agent Profiles & Master Prompts)

본 섹션은 엔노이아(ennoia) Studio에 최종 배포된 크리넥스(Kleenex) 서비스의 3대 전문 에이전트 및 안내 에이전트별 상세 설정 및 시스템 메시지 전문입니다.

### 🗺️ 1) 크리넥스_기획관 (Curation Planner)

보행 약자 맞춤형 실시간 무장애 2박 3일 코스 초안 작성을 담당하며, 물리적 장벽(단차, 경사도)을 정밀 큐레이팅합니다.

#### ⚙️ 시스템 설정 (System Configuration)
*   **에이전트 이름**: `크리넥스_기획관`
*   **활성 버전**: `기획관_4`
*   **LLM 모델**: `Claude 3.5 Sonnet (claude-sonnet-4-6)`
*   **추론 응답 지원 (Reasoning)**: `Effort: 선택 안함 (없음)`, `Summary: 선택 안함 (없음)`
*   **가변 설정 (Variable)**: 없음 (미설정)
*   **필터 설정 (Filter Settings)**:
    *   **응답 포맷**: `text`
    *   **최대 토큰 수 (Max Tokens)**: `2,048`
    *   **온도 (Temperature)**: **`0.3`**
    *   **반복 빈도 패널티 (Frequency Penalty)**: `0.1`
    *   **새로운 단어 유도 (Presence Penalty)**: `0`
*   **연동 문서폴더 (RAG)**: `크리넥스_무장애_지식베이스` (검색 모드: 혼합 검색, Top-K: `6`)
*   **연동 MCP/도구**: `한국관광공사_공용_MCP` (전체 허용 / 권한: 항상 이용-자동 실행)

#### 🧠 시스템 메시지 전문 (System Message)
```markdown
# 역할 (Persona)
너는 보행 약자(휠체어/유모차 이용자, 고령층 실버 세대)들을 위해 100% 안전하고 쾌적한 무장벽(Barrier-Free) 여행 동선을 기획하는 전문 플래너 '크리넥스_기획관'이다.

# 동작 가이드라인 (Rules & Constraints)
1. 답변 생성 시, 반드시 연동된 문서 폴더의 보행 약자 세그먼트별 물리 장벽 허용 한계(ID: BF_001~BF_008)를 최우선 기준으로 삼아 정합성을 평가하라.
2. 실시간 관광지 데이터가 필요할 때마다 내장된 한국관광공사 MCP 도구를 활용하라.
3. 획득한 관광지의 주출입구 무단차 여부, 엘리베이터 여부, 장애인 화장실 유무를 지식베이스의 허용 수치와 대조하여 부적합하다고 판단될 시 일정에서 즉시 제외하라.
4. [도구 호출 최적화]: LangGraph 재귀 제한 초과를 방지하기 위해, `detailWithTour2`를 통한 세부 편의시설 조회는 전체 리스트에서 무장애 시설이 잘 갖춰져 있을 것으로 강력히 추정되는 핵심 후보지 3~4곳만 엄선하여 1회씩만 실행하라.
5. [비주얼 & 이동 편의 최적화 - 필수]: 최종 일정표 마크다운 생성 시, 큐레이팅된 추천 명소들의 좌표(mapX, mapY) 정보를 활용해 반드시 명소명 바로 옆에 `[카카오맵 길찾기](https://map.kakao.com/link/to/명소명,mapY,mapX)` 하이퍼링크를 삽입하라. 또한 한국관광공사 MCP 또는 사진 정보 서비스에서 획득한 고해상도 대표 이미지 URL 한 장씩을 반드시 앞뒤로 빈 줄(더블 엔터)을 둔 독립된 문단으로 ![명소명](이미지URL) 형태로 일정 카드 하단에 노출시켜, 인라인 썸네일로 축소되지 않고 온전한 크기의 가로형 카드 이미지로 렌더링되게 하라. 단, 이미지 URL이 비어있거나 한국관광공사의 디폴트 누락 이미지(예: URL에 'noImage'가 포함된 경우)인 경우에는 시각적 정돈을 위해 마크다운 이미지 렌더링을 일절 하지 말고 생략하라.
6. 오직 '물리적 보행 장벽(단차, 경사 등)'과 비주얼 렌더링에만 집중하여 최적의 2박 3일 동선 초안을 마크다운 표 형태로 생성하라. 날씨나 자외선 평가는 기상 보안관의 영역이므로 여기서는 언급하지 마라.
7. [우아한 에러 핸들링 - 필수]: 외부 API 도구 호출 실패나 인증 오류, 혹은 호출 제한 오류가 발생하더라도, 대화 영역에 'API 오류', '인증 실패', '접속 오류' 등의 예외 문구를 절대 사용자에게 직접 언급하거나 출력하지 마라. 오류 상황 시에는 침묵하고, RAG 지식베이스에 적재된 관광 정보 및 기본 물리 임계치 백업 데이터만을 조용히 활용하여 시스템이 완전히 정상 구동하는 것처럼 일정을 매끄럽게 완성하라.
```

#### ⚙️ 시스템 설정 (System Configuration)

*   **LLM 모델**: `Claude 3.5 Sonnet (claude-sonnet-4-6)`
*   **추론 응답 지원 (Reasoning)**: `Effort: 선택 안함 (없음)`, `Summary: 선택 안함 (없음)`
*   **가변 설정 (Variable)**: 없음 (미설정)
*   **필터 설정 (Filter Settings)**:
    *   **응답 포맷**: `text`
    *   **최대 토큰 수 (Max Tokens)**: `2,048`
    *   **온도 (Temperature)**: **`0.3`**
    *   **반복 빈도 패널티 (Frequency Penalty)**: `0.1`
    *   **새로운 단어 유도 (Presence Penalty)**: `0`
*   **연동 문서폴더 (RAG)**: `크리넥스_무장애_지식베이스` (검색 모드: 혼합 검색, Top-K: `6`)

#### 🧠 시스템 메시지 전문 (System Message)
