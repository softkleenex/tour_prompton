## 참가자 문의 종합 FAQ 가이드북

본 가이드북은 이메일 문의 및 디스코드 실시간 기술 소통 채널을 통해 접수된 참가자들의 주요 애로사항을 취합하여 작성되었습니다. 참가자분들께서는 개발 착수 전 본 내용을 필독하시어 원활한 공모전 진행을 도모하시기 바랍니다.

### 1. 계정 가입 및 권한/접속 오류

#### Q. 메일로 받은 초대 링크를 클릭했는데 "더 이상 유효하지 않은 코드" 또는 로그인 후 "승인 대기(권한 없음)" 창이 뜹니다.
A. 두 가지 원인일 수 있습니다. 아래 내용을 순서대로 확인해 주세요.

1. **링크 주소 맨 끝 확인 ( 대괄호 `]` 제거 )**
   - 발송된 메일 본문 중 URL 끝에 있는 대괄호(`]`)가 주소창에 함께 인식되어 오류가 나는 경우가 있습니다. 인터넷 주소창 맨 끝에 `]` 문자가 포함되어 있다면, 이를 지우고 다시 접속해 보시기 바랍니다.
2. **대괄호를 지워도 접속이 안 되는 경우**
   - **케이스 A: "잘못된 접근" 또는 "더 이상 유효하지 않은 코드"라고 뜨는 경우**
     - **원인:** 발송 과정에서 시스템 오류 등으로 인해 잘못된 링크가 전달되었을 가능성이 높습니다.
     - **조치:** `[팀명 / 팀장명 / 가입 이메일]`을 기재하여 운영사무국 이메일([gongmo@stunning.kr](mailto:gongmo@stunning.kr))로 보내주시면 확인 후 올바른 초대 링크를 다시 발급해 드립니다.
   - **케이스 B: 로그인 후 "승인 대기(권한 없음)" 화면만 반복되는 경우 ★가장 빠른 해결 방법★**
     - **원인:** 공식 초대 링크를 통하지 않고 사이트에 미리 회원가입을 하여 권한이 연동되지 않은 상태입니다. 이 경우는 링크를 새로 받으셔도 해결되지 않으며, 운영진이 수동으로 승인 처리를 해야 합니다.
     - **조치:** 디스코드 채널 내 **[운영/기술 지원]** 스태프에게 디스코드 DM으로 `[팀명 / 팀장명 / 가입 이메일]`을 보내주시면 가장 신속하게 수동 승인 처리가 가능합니다. (디스코드 이용이 어려우신 경우에 한해 운영사무국 이메일로 요청 부탁드립니다.)

> [!IMPORTANT]
> **수동 승인 요청 시 기재 사항**
> - 팀명: 공모전 등록 팀명
> - 팀장명: 대표 참가자 성명
> - 가입 이메일: 엔노이아 플랫폼 가입 계정 이메일

#### Q. 팀원들도 각각 다 가입해야 하나요? 가입 시 '소속'란에는 무엇을 적나요?
A. 팀장님께 발송된 동일한 초대 링크로 팀원 모두 각각 개별 가입하여 공동 작업 공간(Studio)을 사용할 수 있습니다.
- **소속 입력란:** 학교나 직장명 대신, 원활한 명단 매칭 및 권한 부여를 위해 반드시 공모전에 등록한 **`[팀명]`을 정확히 입력**해 주세요.

### 2. 엔노이아(Ennoia) 툴 사용 및 기술 가이드

#### Q. 앱 만들기 단계에서 에이전트를 연결하려는데 "에이전트 없음"으로 뜹니다. 분명히 배포(운영중) 완료했는데요?
A. 엔노이아 스튜디오의 구조상, 에이전트 내에 **'시스템 메시지'와 '도구' 사용 정의만 저장되어 있어야 앱으로 빌드할 수 있습니다.**
- **원인:** 에이전트 테스트 공간에 유저 메시지나 어시스턴트 메시지 등의 **대화 기록(채팅 로그)이 남아있는 상태**로 배포하면 앱에서 불러올 수 없습니다.
- **조치:** 대화 기록을 완전히 지우고 다시 저장 및 배포(Refresh)한 뒤 시도해 보시기 바랍니다.

> [!TIP]
> **대화 기록 초기화 방법**
> 1. 에이전트 설계 화면 오른쪽 테스트 창에서 우측 상단 혹은 하단의 **'대화 초기화(Clear)'** 버튼 클릭
> 2. 대화 기록이 전혀 없는 순수 템플릿 상태에서 **'저장 및 배포'** 진행
> 3. 앱 만들기 화면으로 돌아와 새로고침 후 에이전트 목록 확인

#### Q. 앱 만들기 탭에 에이전트, 멀티에이전트, 커스텀 에이전트, 특화 UI가 있는데 다 써도 되나요?
A. 본 공모전 환경에서는 **"에이전트" 탭과 "멀티 에이전트" 탭 두 가지만 사용하여 결과물을 제작**해 주셔야 합니다.
- 커스텀 에이전트와 특화 UI는 이번 대회 범위에서 다루기 어렵거나 제한되므로 이용을 지양해 주시기 바랍니다.

#### Q. 에이전트와 테스트 대화를 나누고 새로고침을 하면 대화 내용이 왜 전부 사라지나요?
A. 엔노이아 스튜디오(Studio)는 일반적인 ChatGPT 같은 완제품 챗봇 서비스 공간이 아니라 **에이전트를 설계/제작하는 개발 뼈대(IDE) 공간**이기 때문입니다.
- 구조 설계를 완료한 뒤 **'앱(App)'으로 최종 빌드하여 연동한 화면 공간**에서는 새로고침을 해도 대화 내용이 자동으로 정상 유지 및 저장됩니다.

#### Q. 엔노이아에서 음성 출력(TTS)이나 파일 생성 기능을 구현할 수 있나요?
A. 안타깝지만 현재 공모전에 제공되는 엔노이아 버전 내 자체 기능으로는 **음성 출력(TTS) 및 직접적인 파일 생성/다운로드 기능은 기술적으로 지원하지 않습니다.**

### 3. 데이터 활용 및 API 연동 범위 규정

#### Q. 한국관광공사 MCP(Model Context Protocol)를 쓰려는데 인증키(Key)가 필요하다고 합니다. 어디서 받나요?
A. 한국관광공사 API Key는 **공공데이터포털([data.go.kr](https://www.data.go.kr))에서 참가자가 직접 활용 신청을 하여 발급**받으셔야 합니다.
- **참고 문서:** 제공해 드린 노션 대시보드 내 `[2026 관광 데이터 활용 공모전 - OpenAPI 매뉴얼 > 한국관광공사_OpenAPI 활용신청방법_매뉴얼]` 파일에 상세히 안내되어 있으니 매뉴얼 가이드를 반드시 다운로드하여 참고해 주세요.

#### Q. "OpenAPI 형태만 인정, 파일데이터 활용 불인정" 조항이 있던데, 캐싱이나 외부 파일 활용도 아예 안 되나요?
A. 데이터 활용 기준은 아래와 같이 명확히 규정됩니다.

| 데이터 구분 | 원칙 | 세부 인정 범위 |
| :--- | :--- | :--- |
| **한국관광공사 데이터** | **실시간 OpenAPI 호출 필수** | - 시스템 안정성을 위한 **일시적 캐싱 등 보조적 활용은 내부 검증 단계에서 소명 시 인정** 가능<br>- API 호출 없이 정적 파일 데이터(CSV, JSON 등)만 탑재하여 구동하는 방식은 **불인정** |
| **외부 기관 / 민간 데이터** | **자율적 혼합 활용 가능** | - 한국관광공사 필수 데이터 외에 보조적으로 활용하는 타 기관(예: 기상청, 식약처 등)이나 민간 데이터(예: 네이버/카카오 지도 API 등)는 **파일 형태(CSV/JSON 등 RAG 연동) 및 타 API 형태로도 자유롭게 활용 가능**<br>- 단, 메인 서비스 핵심 데이터는 반드시 한국관광공사의 공공데이터여야 함 |

### 4. 공모전 부문 차이 및 최종 제출물 형태

#### Q. 이 공모전의 최종 제출물은 정확히 어떤 형태인가요? 별도의 프론트엔드 웹 화면까지 다 만들어야 하나요?
A. 본 생성형 AI 활용 관광 프롬프톤 부문은 **‘엔노이아 툴 및 한국관광공사 OpenAPI를 활용하여 제작된 대화형 서비스 앱’ 빌드를 최종 목적**으로 합니다.
- **프론트엔드 구축 의무 여부:** 자체적인 프론트엔드 화면 구축이나 독립된 웹/앱 서비스를 직접 코딩하여 구현하는 것은 **참가자 자율 선택 사항**이며, 필수 제출 의무는 없습니다. (미제출로 인한 감점 및 불이익 없음)
- **예선 심사 제출물:** 예선 심사 기준 및 세부 제출 항목은 노션 대시보드에 게시된 **'서비스 제출 안내'**를 참고하여 준비해 주시기 바랍니다.

#### Q. [②-1 프롬프톤 부문]과 [②-2 웹·앱 구현 부문]의 차이는 무엇이고, 중복 참여가 가능한가요?
A. 두 부문은 대회 목적과 개발 방식에서 다음과 같은 차이가 있습니다.

* **②-1 생성형 AI 활용 관광 프롬프톤 부문 (본 대회)**
  - **목적:** 한국관광공사 OpenAPI 및 생성형 AI 기술 기반의 창의적인 관광 서비스 도출
  - **개발 방식:** 참가자들의 원활한 서비스 구현을 위해 전용 개발 툴(엔노이아) 제공
  - **대상:** 개발 역량이 없는 비개발자를 포함하여 누구나 제한 없이 참여 가능
* **②-2 웹·앱 구현 부문 (7월 개최 예정)**
  - **목적:** 프롬프톤에서 발굴된 우수 아이디어 기반의 주제 중 하나를 선택하여, 한국관광공사 OpenAPI를 필수로 연동·활용하여 실질적인 서비스를 구현하고 기술적으로 고도화
* **중복 참여 여부:**
  - **불가:** 동일한 아이디어/동일 서비스로 두 부문에 중복 지원하는 것은 불가능합니다.
