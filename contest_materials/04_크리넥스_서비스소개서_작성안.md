## 📐 SLIDE 3: 서비스 핵심 아키텍처 (System Canvas Architecture)
*   **슬라이드 제목**: **LangGraph 기반의 실시간 3단계 오케스트레이션 엔진**
*   **핵심 기술 논리 (Linear Pipeline)**:
    *   단일 챗봇의 구조적 한계와 할루시네이션(환각)을 극복하기 위해 역할을 정교하게 고립시킨 **3개의 전문 에이전트 직렬 파이프라인** 구축.
*   **동작 매커니즘**:
    ```
    [1. Start 노드] ➡️ [2. 크리넥스_기상_보안관] ➡️ [3. 크리넥스_기획관] ➡️ [4. 크리넥스_비상_우회관] ➡️ [5. User Approval]
    ```
    1.  **시작**: 전역 변수(user_needs, weather_status 등) 선언 및 질문 수신.
    2.  **기상_보안관**: 실시간 기상청 API를 연동하여 기상 이변 등급(`NORMAL/CAUTION/EMERGENCY`) 판정 및 저장.
    3.  **기획관**: 실시간 한국관광공사 MCP API를 연동하여 무장애 관광지 기반의 1차 일정표 초안 빌딩.
    4.  **비상_우회관**: 기상 보안관의 결과가 `CAUTION/EMERGENCY`일 경우, 기획관이 짠 야외 일정을 100% 에어컨과 무단차가 확보된 고품격 실내 코스(MMCA, 서울공예박물관 등)로 자동 대체(Swap).
    5.  **사용자 승인**: 안전성 정합성 검증 리포트를 보여주고 최종 사용자 승인을 얻어 뽀송 일정 출력.

## 🔌 SLIDE 4: 실시간 공공 OpenAPI 연동 명세 (Live APIs Used)
*   **슬라이드 제목**: **이종(異種) 실시간 데이터의 유기적 융합 및 매핑**
*   **핵심 연동 API 및 활용처**:
    1.  **한국관광공사 국문 관광정보 서비스_GW**: 전국 관광지 및 식당 원천 정보 조회.
    2.  **한국관광공사 무장애 여행 정보**: 주출입구 경사로, 턱의 유무, 장애인 화장실 규격, 휠체어 대여소 대수 실시간 조회.
    3.  **한국관광공사 관광 사진 정보**: 대화창에 고해상도 경관 이미지를 동적으로 렌더링하기 위한 데이터 조회.
    4.  **대한민국 기상청 생활기상지수 조회 서비스V4**: 자외선지수(`getUVIdxV4`) 및 체감온도지수(`getSenTaIdxV4`) 실시간 호출.
*   **기술적 차별성**:
    *   정형화되지 않은 텍스트 매칭이 아닌, 각 관광지의 **실제 고유 코드(contentID)**와 기상청 행정구역코드를 직접 대조하여 정밀한 데이터를 연산 및 추론함.

## 📚 SLIDE 5: 정형 CSV RAG 지식베이스 설계 (Structured Knowledge Base)
*   **슬라이드 제목**: **할루시네이션 Zero! 엔노이아 맞춤형 정형 CSV RAG**
*   **도입 배경**:
    *   일반적인 PDF 임베딩 RAG는 텍스트를 무작위로 쪼개어 수치적 임계치 매칭 시 심각한 환각 에러 발생.
*   **해결책**:
    *   엔노이아의 엄격한 **CSV 파일 규칙(ID 선언, 영문/숫자/밑줄 헤더, 큰따옴표 래핑)**에 완벽히 부합하는 정형 데이터베이스 구축 및 데이터셋 다각화.
*   **적용 데이터셋 스펙 (`knto_barrier_free_kb.csv` - 총 14개 클래스 완비)**:
    *   `BF_001` (수동휠체어 단독): 문턱 1.5cm 이하, 경사 4.7도 이하.
    *   `BF_006` (시각장애인 배려): 점자블록 및 음성유도기 필수, 오디오 가이드/문화해설사 투어 연계.
    *   `BF_008` (임산부 배려): 연속 보행 30분 미만, 무단차 및 수유실 등 휴게 시설 연계.
    *   `WE_002` (자외선 위험): 자외선 지수 8 이상 시 야외 활동 전면 금지 및 실내 전환.
    *   `WE_003` (체감온도 폭염): 32도 이상 시 15분 보행 제한 및 대형 복합몰 휴식 1시간 강제.
    *   `WE_004` (미세먼지 위험): PM2.5 75 ㎍/㎥ 이상 시 야외 전면 취소, 1급 환기 실내 코스 스왑.

## 🛠️ SLIDE 6: 핵심 개발 시 시행착오 및 기술적 극복 (Trials & Triumphs)
*   **슬라이드 제목**: **더 견고하고 안정적인 시스템을 향한 엔지니어링 집념**
*   **시행착오 A**: LangGraph 재귀 제한 초과 에러 (`Recursion limit of 25 reached`)
    *   *현상*: AI가 50개 관광지의 무장애 정보를 하나하나 API 루프를 돌며 전수조사하다가 한도인 25단계를 초과해 시스템 중단 발생.
    *   *극복*: 시스템 프롬프트에 **'도구 호출 예산 제약(Throttling)' 규칙**을 주입하여, 전체 리스트 중 가장 유력한 핵심 후보지 3~4곳만 엄선하여 상세 조회하도록 설계하여 응답 속도 비약적 향상 및 에러 원천 차단.
*   **시행착오 B**: 복사-붙여넣기 시 미세 공백으로 인한 API 커넥터 URL 검증 에러
    *   *현상*: 엔노이아 URL 인풋창에 미세한 줄바꿈이나 공백이 유입되어 API 호출 실패 발생.
    *   *극복*: URL 문자열 뒤의 보이지 않는 공백을 수동 바인딩 정합성 검증 단계에서 백스페이스 정밀 튜닝을 통해 해결.

## ♿ SLIDE 7: 실사용 시나리오 A - 날씨가 안전할 때 (NORMAL Scenario)
*   **슬라이드 제목**: **경관과 역사 문화를 듬뿍 만끽하는 평탄한 뽀송 일정**
*   **작동 로직**:
    *   기상청 API가 자외선 지수 보통(NORMAL) 등급 반환 ➡️ 기획관이 평탄 보행로 위주의 실외/실내 조화 코스 설계 ➡️ 비상 우회관이 날씨 정상이므로 코스를 수정하지 않고 즉시 통과.
*   **생성 일정의 핵심**:
    *   경복궁 광화문 경사로 진입 및 근정전 내부 포장 동선(박석 우회 안내).
    *   쌈지길 내부 경사로 전층 관람 동선.
    *   무장애 등록 숙박시설(호텔 더 디자이너스 종로점) 숙소 추천.

## 🚨 SLIDE 8: 실사용 시나리오 B - 날씨가 위험할 때 (EMERGENCY Scenario)
*   **슬라이드 제목**: **기상 경보 발령 시 100% 안전한 무단차 실내 동선 자동 치환**
*   **작동 로직**:
    *   기상청 API가 자외선 매우높음(8 이상, EMERGENCY) 또는 체감온도 32도 이상 반환 ➡️ 기상 보안관이 즉각 위험 상태 저장 ➡️ 비상 우회관이 발동되어 기획관이 짠 야외 일정(청계천 도보 등)을 즉시 포착 ➡️ 에어컨 냉방 및 무단차 휠체어 리프트가 완비된 **'MMCA 서울 실내 전시', '대한민국역사박물관', '서울공예박물관'**으로 동선을 강제 스왑(Swap)하여 제공.
*   **효과**:
    *   폭염 및 직사광선 아래에 보행 약자를 노출시키지 않고, **안전이 보장된 명품 실내 문화 여행**으로 동선을 완전 자동 보정.
