## 🎨 1. 서비스 아키텍처 및 데이터 흐름도 (System Architecture)

엔노이아 캔버스의 단일 출력선 제한 및 노드 간 충돌을 방지하며, 대화 이력 공유(Context Sharing)의 장점을 극대화한 최고 수준의 랭그래프(LangGraph) 연산 흐름을 가집니다.

## 🔌 2. 실시간 공공 OpenAPI 연동 명세 (Live APIs Integration)

크리넥스는 이종(異종) 실시간 공공 데이터를 단일 프롬프트 상에서 연쇄적으로 바인딩 및 파싱하여 실재하는 최신 인프라 정보에 근거한 추론을 수행합니다.

### 🌐 한국관광공사 OpenAPI (내장 MCP 도구 활용)
1.  **국문 관광정보 서비스_GW**: 전국 관광지 및 음식점 원천 좌표 데이터 수집.
2.  **무장애 여행 정보**: 각 시설의 경사로 유무, 단차 높이, 승강기 유무, 장애인 화장실 규격, 휠체어 대여소 대수 실시간 조회.
3.  **관광사진 정보**: 큐레이팅된 명소를 시각화하기 위해 고해상도 경관 이미지 URL 동적 로드.

### ☀️ 대한민국 기상청 OpenAPI (API 커넥터 V4 Swagger 이식)
1.  **생활기상지수 조회 서비스V4 - 자외선지수 (`getUVIdxV4`)**: 보행약자의 야외 활동에 해로운 실시간 자외선 노출 강도 조회.
2.  **생활기상지수 조회 서비스V4 - 체감온도지수 (`getSenTaIdxV4`)**: 폭염/열사병 위험에 노출되기 쉬운 고령층 및 보행약자를 위한 온열 안전도 평가.

비정형 텍스트 PDF 임베딩 RAG가 범하는 청크 슬라이싱 및 수치 왜곡 에러를 원천 제거하기 위해, 엔노이아의 엄격한 **CSV 파일 규칙(ID 고유 식별자, 특수문자 없는 헤더, 문자열 큰따옴표 감싸기)**에 완벽히 부합하는 정형 지식베이스 데이터셋(`05_크리넥스_무장애_지식베이스.csv`)을 구축했습니다.

*   `BF_001` (수동휠체어_단독): 최대 허용 문턱 높이 **1.5cm 이하**, 보행로 경사도 **4.7도 이하** 제한.
*   `BF_003` (전동휠체어): 최대 허용 문턱 높이 **2.0cm 이하**, 보행로 경사도 **8.1도 이하** 제한, 전동 급속 충전기 연계 권장.
*   `WE_002` (자외선_위험): 실시간 자외선 지수 **8 이상** 시 야외 활동 전면 통제 및 100% 실내 대피 코스 유도.
*   `WE_003` (체감온도_폭염): 체감온도 **32도 이상** 시 야외 보행 15분 미만 제한 및 에어컨이 완비된 대형 복합몰 휴식 시간 1시간 필수 배치.

## 🛠️ 5. 핵심 개발 시 시행착오 및 극복 (Trials & Triumphs)

1.  **LangGraph 재귀 제한 25회 돌파 (`Recursion Limit Error`)**
    *   *원인*: 큐레이팅 에이전트가 50개 관광지의 상세 편의시설 정보를 무한 루프를 돌며 API 전수조사하다가 한도인 25단계를 초과해 시스템 락 발생.
    *   *해결*: 에이전트 뇌세포에 **'도구 호출 예산 제약(Throttling) 프롬프팅'**을 강제하여, 전체 관광지 중 무장애 보행이 유력해 보이는 **가장 핵심적인 후보지 3~4곳만 엄선하여 상세 조회(detailWithTour2)**하게 교정함으로써 연산 효율 500% 개선 및 에러 원천 차단.
2.  **API 커넥터 복사-붙여넣기 줄바꿈 에러**
    *   *원인*: 기상청 OpenAPI 연동 시 복사된 문자열 끝부분에 눈에 보이지 않는 공백과 줄바꿈 문자(\n)가 포함되어 "올바른 URL 형식이 아닙니다" 에러 유입.
    *   *해결*: 인풋창에 문자열을 붙여넣은 뒤, 키보드 백스페이스(지우기)를 통해 마지막 특수문자 `}` 바로 옆에 강제 밀착 튜닝함으로써 URL 검증 통과 성공.
